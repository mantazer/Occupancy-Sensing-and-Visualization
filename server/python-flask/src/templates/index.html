About

The timeline leading to the Rice Hall Occupancy project.

  • November 2011

    Rice Hall Opens

    After almost two and a half years of construction, Rice Hall finally opens its doors to the Engineering School and greater U.Va Community. Students in the E-School quickly discover the greatness of Einstein Bagels and the number of study spaces.

  • March 2013

    A Problem is Discovered

    Despite the number of study spaces, finding an open spot took a lot of physical time and effort. Four students (Jonathon Bloncheck, Vinay Dandekar, Shiv Sinha, and Anish Simhal) decided to tackle this problem using an embedded system. Although they were unable to finish the project due to time, the four discovered that the problem was solvable.

  • August 2014

    The Batton is Passed

    Realizing the usefulness of this project, a different group of four students took the Rice Hall initiative under their wings as part of their capstone project. Although the idea came from the previous team, a different approach (using varying development boards and protocols) was taken in hopes of making the system more expandable and efficient.

  • December 2014

    A System is Born

    At the end of the semester, a functional system emerged. Among the most important characteristics of the finished system is the ease at which the system can be expanded. The hope is that, with support from the U.Va Engineering School and Rice Hall, the system will be integrated throughout the building.

PIR Sensor

Detecting Motion


We selected Parallax's PIR sensor (Rev B) for their extensive field of view, ideal operating voltage conditions, and ease of integration. Parallax's PIR sensors can detect motion up to 30 feet away and have an approximately 90 degree field of view. Additionally, they operate in a voltage range of 3V to 6V, which makes them ideal for integrating as digital inputs with CC3200. We are currently operating our PIR sensors at 4.5V by using 3 AAA batteries in a series formation. Whenever motion is detected by PIR sensors, they output a 4.5V signal which is passed on as a digital input to the CC3200.

CC3200 Launchpad

Embedded Board with Internet Stack

The SimpleLink Wi-Fi CC3200 LaunchPad evaluation kit is a development platform for the CC3200 wireless microcontroller (MCU), the industryâ€™s first single-chip programmable MCU with built-in Wi-Fi connectivity. The board features on-board emulation using FTDI device and includes sensors for a full out-of-the-box experience. This board can be directly connected to a PC using software development platforms including CCS and IAR.

We chose to use the CC3200 because our project requires sending environmental motion data from a peripheral sensor to a web server via HTTP across WiFi. The CC3200 takes care of the internet protocol stack layers below the application layer, which allowed us to focus on integrating the PIR sensor with the CC3200 and sending data to the web server rather than having to focus on the low-level connections and handshakes needed to accomplish this. It also allowed us to spend time developing a website and mobile application for the user to view the data in real time.

Amazon Web Services Server

Central Database and Server









Amazon Web Services is the premier cloud service, with estimates that approximately 1% of all internet traffic runs through Amazon's servers. Because of the team's past experience using Amazon Web Services, as well as its price and convenience, we decided to setup both our web and application server on an instance in Amazon's datacenters.

As with any significant application, a backend server is a necessity. Since we did not have quick access to the servers from the Electrical Engineering Department within U.Va (and since these servers often have to be programmed in PHP), we decided to use Amazon Web Services.
